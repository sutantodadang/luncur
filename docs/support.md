# Support luncur

luncur is free and open source (AGPL-3.0). If it replaced a PaaS bill for your
side project, agency, or startup, here is how to keep it healthy — from a
one-click star to a monthly sponsorship.

## Sponsor development

[Sponsor on GitHub ❤](https://github.com/sponsors/sutantodadang){ .md-button .md-button--primary }

Every luncur feature ships with a written design spec, a reviewed
implementation plan, and tests that run without a cluster or network — 230+ of
them. That discipline is why a self-hosted tool that manages your servers,
secrets, and TLS certificates is safe to trust. It also takes time, and time is
what sponsorship buys.

**What your sponsorship funds:**

- **Development time on the roadmap.** Sponsors help set the order — open an
  issue and say what you need.
- **Real-world test infrastructure.** VPSes, domains, and DNS zones across
  Cloudflare / Route53 / RFC2136 so wildcard-cert issuance is tested against
  real providers, not mocks.
- **Fast security responses and dependable maintenance** — the boring,
  essential work that keeps a self-hosted tool trustworthy.

**Who should sponsor:** if luncur is standing in for a managed PaaS, consider
sending a slice of the difference. One month of a hobby-dyno bill funds real
features here.

## Other ways to help

Not everyone can sponsor — these help just as much:

- **⭐ Star the repo.** It genuinely helps others find luncur:
  [github.com/sutantodadang/luncur](https://github.com/sutantodadang/luncur).
- **Report bugs and request features.** Open a
  [GitHub issue](https://github.com/sutantodadang/luncur/issues). For a
  feature, one sentence on the use case beats a long spec.
- **Contribute code.** The dev loop is `go test ./...` — no cluster needed.
  See [CONTRIBUTING.md](https://github.com/sutantodadang/luncur/blob/main/CONTRIBUTING.md).
- **Report security issues privately.** Please don't open a public issue for a
  vulnerability — use
  [private reporting](https://github.com/sutantodadang/luncur/security/advisories/new)
  or email `sutantodadang@gmail.com` with `[luncur security]` in the subject.
  See the [security policy](https://github.com/sutantodadang/luncur/blob/main/SECURITY.md).
- **Spread the word.** A blog post, a tweet, or a "we run luncur" mention helps
  more than you'd think.

## Commercial licensing

luncur is AGPL-3.0: use it, self-host it, modify it, fork it. The AGPL asks one
thing — if you run a **modified** luncur as a service for others, share those
modifications back.

Need different terms? For proprietary embedding, OEM redistribution, or
offering luncur as a managed service without AGPL obligations, a separate
**commercial license** is available from the copyright holder. Open a GitHub
issue or reach out directly — commercial licensing also funds ongoing
development.
